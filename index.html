Frågor och svar

  1. Github flow - Vad är det? Hur använder man det? Vilka viktiga delar är inkluderade i begreppet?

    2. Svar:

    Gitflow är en fast arbetsrutin som man följer för att synka sina lokala filer med remote filer. Vi gör detta genom att sätta den utvalda datan vi har på våran dators minne i en “paket” som vi kan kalla för git. Paketet kan nu skickas, eller pushas, vidare till github, en remote repository (ett avlägset förvar).

    Sedan kan ytterligare steg tillsättas om projektet involverar flera personer än bara sig själv. Då ska man även synka mot en main branch genom att göra pull requests, vänta på att ens kamrater gett deras godkännande, merge:a ens branch med github-repot och pull:a den lokalt.

    Jag kan ge ett exempel med ett projekt där flera är involverade så går vi igenom alla stegen.

    Någon har gjort ett repo på github och bjudit in mig som collaborator. Jag accepterar. Personen har även gjort issues, av vilket minst en av dem har tilldelats mig. Jag klonar repot och skapar en lokal branch där jag gör min del av arbetet.

    Jag använder nu hela tiden den individuella gitflow:en. Jag lägger till ändringar och sparar dem i filen på datorn. När jag känner mig nöjd med en ändring kan jag sätta det i git med git add eller stage:a ändringarna. När jag känner mig nöjd med en version av min uppgift kan jag göra git commit så “tas en bild” av den versionen och ligger nu sparad i git-historiken som en “snapshot” jag nu hela tiden kan nå. “Snapshot:en” kan nu pushas upp till repot på github med git push så att alla ser min branch.

    Jag kan nu begära en pull request och samtidigt göra en uppdatering i issues-tråden medans jag väntar på feedback och eventuellt godkännande från kollaboratörer för att kunna gå vidare med en merge av min egna branch med den huvudsakliga/main branch:en dit allas "approved" arbetsinsatser läggs till.

  2. Flexbox - Vad är det? Hur fungerar det?

    3. Svar:

    Flexbox är en declaration man sätter på ett element genom .CSS. Det som då händer är att elementet nu ses som en flex-container. Detta betyder att dess direkta barn blir flex-items.

    Nu tillgängliggörs specifika flex-properties som man kan utnyttja. Det finns dem som man kan utnytta på SJÄLVA containern genom att sätta dem i elementet i .CSS dit man lagt "display: flex;". Dessa är flex-flow, flex-direction, flex-wrap, justify-content, align-items, och align-content. Sedan finns det dem flex-properties som man kan sätta på en flex-container:s flex-items i deras declaration block i .CSS såsom flex-shrink, flex-grow, flex-basis, align-self, och order. Alla dessa properties bestämmer hur flex-items ligger i relation till varandra i flex-containern och var dem ligger på main-axis och cross-axis.

  3. DOM (Document Object Model) - Vad är det? Hur manipulerar man DOM?

    4. Svar:

    DOM är ett sätt att rekontextualisera HTML-dokument i ett trädliknande ramverk. Dess olika beståndsdelar omdefinieras. Element blir nu objekt. Objektets innehåll (attribut och textinnehåll) blir nodes och objektet själv är en node. Nodes kan vi nå och antingen ändra, ta bort, eller skapa och sätta in.

    Och hur gör man det? Man gör det med Javascript. Man skapar en .js-fil och refererar den i sin script-tagg i HTML-filen. I själva .js-filen kan jag nå alla nodes i DOM eller skapa dem med hjälp av olika metoder och properties som jag kallar på. Man kan även spara dem i variabler och få fram deras values.

  4. Fin kod - Vad menas med det begreppet? Varför är det viktigt?

    5. Svar:

    Fin kod betyder strukturerad, omtänksam, och hållbar kod. Man ska kunna komma tillbaks om några år och snabbt kunna komma in i vad allt det olika innehållet har för roll i källkoden. Sedan ska andra också kunna läsa och tyda koden.

    Det finns några saker som man kan göra för att förebygga oläsbar kod. Man kan använda sig av kommentarer för att både kommentera dem olika delarna men kan också utnyttja kommentarer i form av orienteringsundertitlar som man sätter ovanför olika avsnitt/indelningar. Class- och id-attributen kan vara till stor hjälp. Också namnet på en funktion eller variabel. Sist tänkar jag indrag av de olika elementen och spacing mellan stycken för att få ordning på dem.

    Några saker man kan undvika är att att förklara med kommentarer syftet hos variabler eller funktioner. Deras namn borde kunna stå för sig själva. Var också försiktig med programmeringsfackord (ex. datatyper) i namn; det kan vara vilseledande. Sist, när namn blir tydliga så kan de bli långa. Undvik därför onödiga ord som kanske ger grammatik men som inte informerar.

  5. Ramverk - Inom webbutveckling använder man begreppet ramverk, vad är det? Beskriv vad det är och berätta lite mer om två valfria.

    6. Svar:

    Ett ramverk är färdig, återanvändbar server-side kod som man kan importera och utnyttja i sitt eget projekt så att man slipper skriva all sin kod från scratch.

    Bootstrap är ett ramverk för front-end utveckling, alltså kan man dra nytta av den för att smidigt ge sin sida ett mer standariserat applikationsutseende. Det finns ett annat ramverk kallad Vue.js också. Då däremot Bootstrap har med app-liknande utseende att göra, kan Vue hjälpa till med app-liknande beteende. Det finns också Electron som är ett allmänt ramverk som uttnyttjar alla tre webbteknologier (HTML, CSS och Javascript) som sätts i bruk för att hjälpa till med funktionaliteten hos desktop-applikationer.
